# BA Specification: Sidebar Collapse/Expand UI Redesign

**Request ID**: dev-20260323-210000
**Created**: 2026-03-23T21:00:00Z

## Goal

Replace the mismatched Ionicons used for sidebar collapse/expand controls with brutalist-style image assets that match the existing design language, and improve the visual hierarchy of the sidebar header icon group.

## Context

The sidebar header in SidebarView.tsx uses a mix of two icon systems:
- **Brutalist image assets** (32x32 PNGs with tintColor) for inbox (Brutalism 27 = starburst) and settings (Brutalism 9 = grid pattern)
- **Ionicons vector icons** for collapse chevron (`chevron-back-outline`, 22px) and new session plus (`add-outline`, 28px)

This creates a jarring visual inconsistency: the brutalist icons have thick, hand-drawn, geometric aesthetics while the Ionicons are thin, uniform-weight system icons. The chevron especially looks out of place -- it reads as a navigation "back" button rather than a sidebar collapse control.

The expand button in SidebarNavigator.tsx (hamburger menu, `menu-outline`, 18px) also uses Ionicons and sits on a small 32x32 surface-colored square, which does not match the brutalist identity.

## Requirements (MoSCoW)

### Must Have
- Replace the collapse chevron (`chevron-back-outline`) in SidebarView.tsx with a brutalist image asset, using the same 32x32 + tintColor pattern as the inbox/settings icons
- Replace the expand hamburger (`menu-outline`) in SidebarNavigator.tsx with a brutalist image asset, same pattern
- Replace the "+" new session button (`add-outline`) in SidebarView.tsx with a brutalist image asset (Brutalism 14 = sparkle wand is a strong candidate -- it conveys "create new" while matching the aesthetic)
- All replacement icons must use `tintColor={theme.colors.header.tint}` for theme consistency
- Maintain existing hitSlop={15} and press handlers

### Should Have
- Improve the expand button styling: instead of a plain surface-colored square, use a subtle border or shadow that matches the sidebar's visual language (e.g., `theme.colors.divider` border, or match the sidebar's own border style)
- Reorder the header icons for better semantic grouping: [inbox] [settings] [new session] ... [collapse] -- the collapse button should be the rightmost icon since it controls the sidebar panel itself, separated from the content-action icons

### Could Have
- Add a hover state (web only) to the expand button for better discoverability
- Animated transition on the collapse icon (e.g., subtle scale on press)

### Won't Have (Non-Goals)
- Changing the sidebar collapse/expand animation or behavior
- Modifying the mobile drawer behavior (this is web/tablet only)
- Creating new custom icon assets (we use existing brutalist set)
- Changing the logo or title area layout

## Edge Cases & Risks

- **Icon readability at 32x32**: Some brutalist icons are detailed and may not read well at small sizes. The chosen icons must be recognizable at 32px with tintColor applied.
- **Dark mode**: All brutalist icons already use tintColor, so dark mode should work. Verify the expand button background works in both themes.
- **No suitable brutalist arrow icon**: The brutalist set does not contain a clear left-arrow or sidebar-panel icon. Alternative approach: use Brutalism 9 (grid) rotated, or Brutalism 22 (cubes) for the expand button since it suggests "panels/layout". Or simply keep a clean Ionicons sidebar icon but style it to match (thicker weight).

## Acceptance Criteria

### AC1: Visual consistency of collapse button
- GIVEN the sidebar is expanded on a tablet/web view
- WHEN the user looks at the sidebar header icon row
- THEN all icons (inbox, settings, collapse, new session) use the same visual style (brutalist image assets at 32x32 with tintColor)

### AC2: Visual consistency of expand button
- GIVEN the sidebar is collapsed on a tablet/web view
- WHEN the expand button is visible in the top-left corner
- THEN the expand button uses a brutalist image asset with styling that feels cohesive with the rest of the UI (not a plain floating square)

### AC3: Functional equivalence
- GIVEN the new icons are in place
- WHEN the user clicks collapse, it collapses the sidebar
- AND WHEN the user clicks expand, it expands the sidebar
- AND WHEN the user clicks new session, it navigates to /new
- THEN all behaviors are identical to the current implementation

### AC4: Icon reordering (should-have)
- GIVEN the sidebar is expanded
- WHEN viewing the header icon row
- THEN the order is: [inbox] [settings] [new session] [collapse] -- with collapse as the rightmost icon
